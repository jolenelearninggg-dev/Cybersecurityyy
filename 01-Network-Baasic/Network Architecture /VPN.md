## VPN (虛擬私人網路) 與網路連線
### 1. VPN 的核心功能與原理
定義： VPN 在公共網路上建立一條安全加密的隧道，讓使用者能安全地存取遠端私人內部網路。

加密路徑： VPN 確保從 您的設備 到 VPN 伺服器 之間的流量是加密的，防止 ISP 或中間竊聽者查看內容。

IP 偽裝： 您的流量會被重新導向，對外顯示的 IP 位址是 VPN 伺服器的公用 IP，藉此隱藏您的真實 IP。

兩種類型：

SSL VPN： 使用網頁瀏覽器連線，無需客戶端軟體。

客戶端型 VPN： 需要專門軟體，提供更廣泛的內部網路存取。

<br>

### 2. 加密與安全性的盲點
全程加密的迷思： VPN 伺服器必須先解密流量，才能知道您的最終目的地。

HTTP 風險： 如果目標網站是 HTTP (非加密)，資料在離開 VPN 伺服器到目標網站的這段路徑上將是未加密的。

供應商風險： 雖然加密協議強大，但 VPN 服務商可能記錄 (Logging) 您的連線活動，因此無法保證絕對隱私。

<br>

### 3. 網路基礎設施 (ISP)
ISP 定義： 網際網路服務供應商 (ISP) 是您連線到全球網路的通道，負責分配您的公用 IP 位址。

IP 目的： 公用 IP 是讓世界上其他網路主機能識別並將資料送回給您的唯一地址。

<br>

### 4. 路由表 (Netstat -rn) 解讀
功能： 路由表是作業系統決定網路封包傳輸路徑的交通規則手冊。

關鍵規則：

預設路由 (0.0.0.0)： 處理所有一般上網、不符合其他規則的流量（通常透過實體網卡 eth0）。

VPN 路由 (10.129.0.0)： 處理所有發往 VPN 內網特定網段的流量（強制通過虛擬網卡 tun0）。

tun0 介面： 是一個虛擬網卡，代表 VPN 加密通道的出口。
