## VM 虛擬化與 NAT 隔離
### 1. 使用 VM 連線惡意網路的原因
隔離目的： 透過硬體虛擬化，將高風險操作限制在一個獨立的沙盒環境中，以保護主作業系統 (Host OS) 的安全。

防止跳脫： 即使 VM 被攻擊者入侵，他們也難以「跳脫」出來攻擊主機系統。

### 2. NAT 模式的隔離機制
NAT 定義： 網路位址轉換，是虛擬化軟體常用的網路模式。

外部視角： VM 對外的連線都會被主機轉換，外部網路永遠只看到主機的公用 IP。

隔離效果： 雖然 NAT 無法隱藏您的公用 IP，但它：

防止外部主動連線： 外部網路無法直接發起連線到 VM 內部的私有 IP。

易於還原： 讓您可以在被感染後，隨時透過快照恢復或直接刪除 VM，而不會危及主機系統。
