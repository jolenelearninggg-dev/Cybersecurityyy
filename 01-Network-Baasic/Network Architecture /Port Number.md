## 💻 網路埠號 (Port Numbers) 基礎與資安原則筆記
### 埠號 (Port) 的功能與定義
網路埠號（Port Number）是 TCP/IP 協定中用來識別網路服務的數字標籤。
>*想像一個 IP 位址是一棟大樓的地址，那麼埠號就是這棟大樓內不同服務所使用的房間號碼。*

IP 與埠號： IP 位址是網路地址。
>*埠號是該地址上特定服務的「門牌號碼」。*

唯一性識別： 透過「IP 位址 + 埠號」的組合，作業系統可以確定傳入的網路封包應該由哪個應用程式（例如網頁伺服器、郵件客戶端、遊戲伺服器）接收和處理。

1. 埠號的基礎概念
定義： 埠號是網路服務的數字標籤 (0-65535)，用於識別應用程式（如 HTTP、SSH）。

標準化： IANA/RFC 標準埠號 (0-1023) 確保了網路軟體的互通性、穩定性和管理便利性（例如 443 埠用於 HTTPS）。

「隱藏埠號」的安全性謬誤
觀念： 透過隱藏來保障安全 (Security through Obscurity) 不是可靠的安全策略。

破解方式： 駭客不會猜埠號，他們會使用 埠號掃描工具 (如 Nmap) 對所有 65535 個埠進行系統性探測，輕易找到非標準埠上的服務。

根本問題： 真正的安全性來自於服務本身的強度（修補漏洞、強密碼），而不是它在哪個埠運行。

3. 實質安全防禦 (代替隱藏)
多層次防禦： 實施防火牆、強大認證（如金鑰登入、MFA）和定期軟體更新。

範圍分類： 埠號範圍從 0 到 65535，通常分為三個區段：

範圍	名稱	說明
0 - 1023	周知埠 (Well-known Ports)	由 IANA 永久指定給最常見的網路服務（如 HTTP、FTP、SSH）。
1024 - 49151	註冊埠 (Registered Ports)	可供使用者程序或非標準化服務使用，但需向 IANA 註冊。
49152 - 65535	動態或私有埠 (Dynamic/Private Ports)	用於用戶端連線時的臨時埠號，連線結束後就會釋放。
