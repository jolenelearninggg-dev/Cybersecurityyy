## 💻 網路埠號 (Port Numbers) 基礎與資安原則筆記 

### 🚪 I. 埠號 (Port) 的核心定義與功能定義： 
 - 埠號是虛擬的點，是網路連線開始與結束的地方。它們是軟體基礎的，由主機作業系統管理。
 - 比喻： 埠號好比一棟房子（遠端系統）上的門或窗戶。如果門窗未正確上鎖（或配置），可能會造成未經授權的存取。
 - 關鍵作用： 埠號與特定的程序或服務相關聯，它允許電腦區分不同的流量類型（例如，SSH 流量和網站請求會流向不同的埠號），即使這些請求是透過相同的網路連線傳輸。
 - IP 與埠號： 透過「IP 位址 + 埠號」的組合，電腦可以精確地將收到的資料導向正確的服務。

<br>
 
### 📊 II. 埠號的分類、協議與範圍
1. 埠號的兩種協議類型 (TCP vs. UDP)
   >TCP (Transmission Control Protocol) 和 UDP (User Datagram Protocol) 是網路傳輸層最重要的兩種協議。它們的根本區別在於對待「資料傳輸可靠性」的態度不同，因此適用於完全不同的網路應用場景。
   
   埠號分為兩大類，每一類都有 $65,535$ 個埠號：
   | 協議 | 類型 | 特點 | 適用場景 |
   | ---- | ----| ---- | ------- |
   | TCP (Transmission Control Protocol) | 連線導向 (Connection-oriented) | 必須先透過 「三次握手 (Handshake)」 建立連線才能傳輸資料。提供可靠性、錯誤修正和保證資料傳輸順序。 | 網頁 (HTTP/HTTPS)、電子郵件 (SMTP)、檔案傳輸 (FTP)。 |
   | UDP (User Datagram Protocol) | 無連線模式 (Connectionless) | 無需握手，直接發送資料。不保證資料傳輸的可靠性或順序。 | 對延遲敏感的應用程式，如 DNS 解析、串流媒體、線上遊戲。 |
   
3. 埠號的三大範圍
   埠號範圍從 0 到 65535，被 IANA (網際網路號碼分配局) 分為三個區段：
   | 範圍 | 名稱 | 說明 |
   | ---- | ----| ---- |
   | 0 - 1023周知埠 | (Well-known Ports) | 由 IANA 永久指定給最常見的網路服務（例如 80/HTTP 和 443/HTTPS）。｜
   | 1024 - 49151 | 註冊埠 (Registered Ports) | 可供使用者程序或非標準化服務使用，但需向 IANA 註冊。 |
   | 49152 - 65535 | 動態/私有埠 (Dynamic/Private Ports) | 用於客戶端連線時的臨時埠號。 |

<br>
   
### 💡 III. 埠號的標準化與資安原則
   1. 標準化的重要性埠號的標準化（如 HTTP 使用 80，HTTPS 使用 443）確保了所有網路設備和應用程式之間的互通性和穩定性。雖然服務可以配置在非標準埠上（例如 Web 應用程式運行在 8080），但通常仍建議使用標準埠以減少配置複雜性。
   2. 「隱藏埠號」的安全性謬誤 (Security through Obscurity)觀念： 認為更改埠號可以讓駭客找不到。破解方式： 駭客會使用 埠號掃描工具 (例如 Nmap) 對所有埠進行系統性探測，輕易就能找到非標準埠上的服務。根本問題： 真正的安全防禦來自於服務本身的強度（修補漏洞、強密碼），而不是埠號的數字。
   3. 實質安全防禦多層次防禦： 實施防火牆、強大認證（如金鑰登入、MFA）和定期軟體更新。
