          |------net-------|------host------|

Class A :   0.xx.xx.xx ~ 127.xx.xx.xx

Class B ⇒ IPv4 地址有 16 個網路字首位，後兩個八位為主機地址

Class C ⇒ IPv4 地址有 24 個網路字首位，後八位為主機地址

** Host ID 全部為 0 $\rightarrow$ Network Address (網段位址) **

1. 網路位址 (Network Address)
 - 特徵： 該網段中 Host ID 全部為 0 的那個 IP。
 - 用途： 這是用來標識（Identify）整個網段的位址。

🌐 為什麼需要它？
1.路由判斷 (Routing Decisions):
 - 路由器（Router）在決定如何轉發數據封包時，並不是看每個單一設備的 IP，而是看封包要前往哪一個網段。
 - 當路由器看到一個目標 Network ID 時，它就知道如何將封包導向正確的網路，即使它不知道該網段內目標設備的確切 Host ID。
2.網路管理與文檔 (Network Management):
 - 網路管理員（Network Administrator）在規劃、配置和記錄網路時，使用 Network Address 來表示一個 IP 區塊，例如：「這個分公司使用 192.168.1.0/24 這個網段。」
 - 總結： 它代表的是「網段本身」，而不是網段內的任一台設備。
2. 廣播位址 (Broadcast Address)特徵： 該網段中 Host ID 全部為 1 的那個 IP。用途： 這是用來向該網段內所有設備同時發送資訊的位址。

📢 為什麼需要它？
1.尋找設備 (Address Resolution - ARP/DHCP):
 - 當一台設備不知道網段內另一台設備的 MAC 位址時（例如使用 ARP 協定時），它會發送一個廣播訊息，詢問：「誰是 $192.168.1.5$？請告訴我你的 MAC 位址！」這個 ARP 請求就是發送到廣播位址。
 - DHCP（動態主機配置協定）客戶端在啟動時，也會向廣播位址發送請求，以找到 DHCP 伺服器並取得 IP 位址。
2.服務發現 (Service Discovery):某些協定允許設備通過廣播來宣布或尋找網段內的特定服務（例如，尋找印表機或特定的伺服器）。
3.效率 (Efficiency):如果需要同時通知網段內所有設備某個資訊，使用單一的廣播位址比發送 $N$ 個單點傳輸（Unicast）封包給 $N$ 個設備要高效得多。總結： 它代表的是「網段內的所有設備」，是一對多通訊的目標位址。
